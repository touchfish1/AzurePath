SNMP 实现方案

三个技术路线对比

A

snmp2 crate（纯 Rust）

原生 Rust SNMP 实现，支持 v1/v2c/v3。

Pros

  • 跨平台，Windows 友好
  • 无需外部依赖
  • 集成管理连接池

Cons

  • 社区较小，更新慢
  • 缺少高抽象 API
B

系统命令（snmpwalk/get）

通过 tokio::process::Command 调用 net-snmp 工具

Pros

  • 功能完整，MIB 解析完善
  • 易于调试

Cons

  • Windows 需要额外安装
  • 进程开销，解析文本输出
C

snmp_mp2 + 自建抽象层

更活跃的纯 Rust crate + 自定义 ASN.1/OID 封装

Pros

  • 纯 Rust，跨平台
  • API 设计更现代

Cons

  • 仍需自己封装业务逻辑
  • 文档不够完善
